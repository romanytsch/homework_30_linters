1. АНАЛИЗ ТЕКУЩЕГО УРОВНЯ ЗРЕЛОСТИ API
================================================================================

Текущий уровень: Level 2 (HTTP Verbs + Resources)

Endpoint'ы:
------------------------------------------------------------
Endpoint        | Метод | Уровень | Характеристика
------------------------------------------------------------
POST /add-room  | POST  | Level 1 | Действие вместо ресурса
GET /room       | GET   | Level 2 | Параметры запроса
POST /booking   | POST  | Level 1 | Действие вместо ресурса
------------------------------------------------------------

Проблемы:
• Названия содержат глаголы (add-, booking)
• Несогласованность (room vs rooms)
• Отсутствует 201 Created


2. ПРЕДЛАГАЕМЫЕ УЛУЧШЕНИЯ (Level 3 - Controllers)
================================================================================

Новая RESTful структура:
------------------------------------------------------------
Старый           | Новый          | Метод | Обоснование
------------------------------------------------------------
POST /add-room   | POST /rooms    | POST  | Коллекция комнат
GET /room        | GET /rooms     | GET   | Список доступных комнат
POST /booking    | POST /bookings | POST  | Коллекция бронирований
------------------------------------------------------------

Изменения параметров:
------------------------------------------------------------
Старое     | Новое       | Обоснование
------------------------------------------------------------
guestNum   | guestsCount | Читаемое camelCase
beds       | bedCount    | Единый стиль
------------------------------------------------------------


3. HTTP СТАТУС КОДЫ
================================================================================
Действие              | Старый | Новый | Обоснование
------------------------------------------------------------
Создание комнаты      | 200    | 201   | Created
Успешное бронирование | 200    | 200   | Корректно
Комната занята        | 409    | 409   | Корректно


4. ОБОСНОВАНИЕ ИЗМЕНЕНИЙ
================================================================================

REST принципы:
------------------------------------------------------------

---------------------------------------------
